# Project History

## Origin

Royal was created to solve a simple problem: most cruise planning tools help people calculate details but fail to answer the real question.

Should I book this cruise or not?

The project evolved from individual calculators into a full Cruise Decision Engine focused on confidence, comparison, and practical decision-making for Royal Caribbean trips.

## Major Milestones

### Initial Launch
- React + Vite single-page application established
- GitHub Pages deployment configured
- Core planning and pricing tools created

### Calculator Expansion
- Deal Evaluator
- Cruise Cost planning
- Dining and package comparison tools
- My Trip Brief and Trip Snapshot

### Product Repositioning
Shifted from "calculator site" to decision engine.

Focus changed from:
"more math"

to:
"clear recommendation and trust"

### Flagship Direction
Defined `Should I Book This Cruise` as the primary product experience.

This became the highest-priority feature and flagship decision page.

### Operational Maturity
- GitHub Issues became the backlog source of truth
- Codex workflow standardized with issue labels
- `AGENTS.md` added for implementation rules
- PR templates, contribution docs, and workflow structure added

## Current Direction

Royal is being built as a trust-first cruise decision platform.

Priority order:
1. Final recommendation engine
2. Snapshot as executive summary hub
3. Homepage conversion path
4. Real examples and lead capture

## Operating Model

- ChatGPT shapes strategy and backlog
- GitHub Issues manage execution
- Codex implements from `codex-ready` issues
- PR review gates protect quality

## Product Principles

- Decision-first
- Mobile-first
- Trust over feature count
- No fake pricing or false certainty
- Useful before impressive

## What We Avoid

- endless calculator sprawl
- generic SEO content
- fake precision
- bloated planner features
- decorative complexity without business value
